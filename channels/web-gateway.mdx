The Web Gateway provides a browser-based chat interface with real-time updates via WebSocket.

## Features

- **Real-time chat**: WebSocket streaming for instant responses
- **Memory browser**: Search and manage your workspace documents
- **Jobs dashboard**: Monitor running and completed jobs
- **Routines**: Schedule and manage automated tasks
- **Extensions**: Install and configure tools
- **Settings**: Configure all options via web UI

## Enabling the Web Gateway

The Web Gateway is enabled by default. To configure:

```bash
export GATEWAY_ENABLED=true
export GATEWAY_HOST=127.0.0.1
export GATEWAY_PORT=3000
export GATEWAY_AUTH_TOKEN=your-secure-token
```

Or during onboarding:
```
Step 6: Channel Configuration
→ Select "Web Gateway"
→ Port: 3000
```

## Accessing the Web UI

Once IronClaw is running:

1. Open `http://127.0.0.1:3000` in your browser
2. Enter the auth token when prompted
3. Start chatting

## Authentication

### Default Behavior

By default, a random auth token is generated at startup and printed in the logs:

```
[2024-01-15T10:30:00Z INFO] Web Gateway: http://127.0.0.1:3000
[2024-01-15T10:30:00Z INFO] Gateway auth token: ic_a1b2c3d4e5f6...
```

### Persistent Token

For a consistent token across restarts:

```bash
export GATEWAY_AUTH_TOKEN="your-secure-token-here"
```

Generate a secure token:
```bash
openssl rand -hex 32
```

### Finding the Token

If you didn't set a persistent token:

```bash
RUST_LOG=ironclaw=info ironclaw run 2>&1 | grep "Gateway auth token"
```

## Network Access

### Localhost Only (Default)

```bash
export GATEWAY_HOST=127.0.0.1
```

Only accessible from the same machine.

### LAN Access

```bash
export GATEWAY_HOST=0.0.0.0
```

Accessible from other devices on your network.

<Warning>
**Security:** When using `0.0.0.0`, ensure you have a strong auth token and consider using a reverse proxy with HTTPS for external access.
</Warning>

## HTTPS / Reverse Proxy

For external access, use a reverse proxy:

### Caddy

```caddy
docs.ironclaw.ai {
  reverse_proxy localhost:3000
}
```

### nginx

```nginx
server {
  listen 443 ssl;
  server_name webg.example.com;

  ssl_certificate /path/to/cert.pem;
  ssl_certificate_key /path/to/key.pem;

  location / {
    proxy_pass http://localhost:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
  }
}
```

### Cloudflare Tunnel

```bash
cloudflared tunnel --url http://localhost:3000
```

## WebSocket Connection

The Web UI uses WebSocket for real-time updates:

- Connects to `ws://host:port/ws`
- Automatically reconnects on disconnect
- Shows connection status indicator

## API Endpoints

The Web Gateway exposes REST endpoints:

| Endpoint | Description |
|----------|-------------|
| `GET /api/status` | Server status |
| `POST /api/chat` | Send message |
| `GET /api/jobs` | List jobs |
| `GET /api/memory` | Search memory |

See [API Reference](/ops/api) for details.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Connection refused" icon="x-circle">
    - Check IronClaw is running: `ironclaw status`
    - Verify correct port: `GATEWAY_PORT`
    - Check firewall: `sudo ufw status`
  </Accordion>

  <Accordion title="Invalid token" icon="key">
    - Copy token exactly from logs
    - Check for trailing spaces
    - Set persistent token: `GATEWAY_AUTH_TOKEN`
  </Accordion>

  <Accordion title="WebSocket disconnects" icon="network">
    - Check network stability
    - Verify no proxy blocking WebSocket
    - Check browser console for errors
  </Accordion>

  <Accordion title="UI not loading" icon="monitor">
    - Clear browser cache
    - Try incognito/private mode
    - Check JavaScript is enabled
  </Accordion>
</AccordionGroup>
