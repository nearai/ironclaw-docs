## Channel Types

IronClaw has two types of channels:

### Built-in Channels

These are compiled into IronClaw itself:

| Channel | Type | Description |
|---------|------|-------------|
| **Terminal UI** | Built-in | Ratatui-based terminal interface |
| **Web Gateway** | Built-in | HTTP server with WebSocket |
| **HTTP Webhook** | Built-in | REST API endpoint |

### WASM Channels

These are separate modules loaded at runtime:

| Channel | Type | Description |
|---------|------|-------------|
| **Telegram** | WASM | Bot integration via Telegram API |
| **Signal** | WASM | Signal-cli integration |

WASM channels are discovered in `~/.ironclaw/channels/` and loaded dynamically.

## Adding Channels

Add channels during onboarding:

```bash
ironclaw onboard --channels-only
```

Or manually configure:

```bash
# Enable Web Gateway
export GATEWAY_ENABLED=true

# Enable HTTP Webhook
export HTTP_HOST=0.0.0.0
export HTTP_PORT=8080

# Start IronClaw
ironclaw run
```

## Security Considerations

Each channel has different security implications:

- **Terminal UI**: Local only, no network exposure
- **Web Gateway**: Configurable auth token, local by default
- **Telegram**: Bot token required, DM pairing for access control
- **HTTP Webhook**: Shared secret validation, rate limited

See [Security](/security) for detailed security guidance.
