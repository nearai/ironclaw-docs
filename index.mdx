## What is IronClaw?

IronClaw is a **secure, self-hosted AI assistant** designed for integrators, system operators, and privacy-conscious users. Unlike cloud-based assistants that send your data to external servers, IronClaw runs on your own hardware—keeping conversations, files, and secrets under your control.

**Key differentiators:**

- **Local-first** — Your data never leaves your machine unless you explicitly configure external LLM providers
- **Security-first architecture** — Multi-layer defense with safety layer, WASM sandbox, Docker isolation, and encrypted secrets
- **Multi-channel access** — Chat via web browser, Telegram, terminal UI, or HTTP webhooks
- **Self-expanding** — Build and install custom tools dynamically without vendor dependency

The architecture shows how user input flows through IronClaw's multi-layer system:

1. **Input Channels** — Web Gateway, Telegram, Terminal UI, HTTP Webhook
2. **IronClaw Core** — Agent Loop, Intent Router, Job Scheduler
3. **Safety Layer** — Sanitizer, Validator, Policy Engine, Leak Detector
4. **Tool System** — Built-in Tools, WASM Sandbox, MCP Servers
5. **Persistent Memory** — Workspace, Documents, Hybrid Search

## Quick Start

Get IronClaw running in under 10 minutes:

<Steps>
  <Step title="Install IronClaw" icon="download">
    Run the install script for your platform:
    ```bash
    curl -fsSL https://install.ironclaw.ai | bash
    ```
    See [installation options](/install) for Docker, VPS, and other methods.
  </Step>

  <Step title="Run the Wizard" icon="settings">
    Launch the interactive setup wizard:
    ```bash
    ironclaw onboard
    ```
    This configures your database, LLM provider, channels, and security settings in 8 steps.
  </Step>

  <Step title="Start IronClaw" icon="play">
    Start the agent:
    ```bash
    ironclaw run
    ```
    If you enabled the Web Gateway, open `http://127.0.0.1:3000` in your browser.
  </Step>
</Steps>

## Security at a Glance

IronClaw's security architecture is designed around **defense in depth**—multiple independent layers that each provide protection:

| Layer | Component | Protection |
|-------|-----------|------------|
| **Safety Layer** | Sanitizer, Validator, Policy, Leak Detector | Prompt injection defense, secret detection, input validation |
| **WASM Sandbox** | wasmtime runtime | Sandboxed tool execution with fuel metering |
| **Docker Sandbox** | Container isolation | Job execution in isolated containers with network proxy |
| **Secrets** | AES-256-GCM encryption | Credentials encrypted at rest, injected at proxy boundary |

Learn more in the [Security Overview](/security).
