Built-in tools are compiled directly into IronClaw. They are always available to the agent â€” no installation required. The agent selects them automatically based on the task at hand.

<Note>
File and shell tools require `ALLOW_LOCAL_TOOLS=true` to be set. They are disabled by default to prevent accidental filesystem or command access in hosted or shared environments.
</Note>

## File Tools

| Tool | Description | Config required |
|------|-------------|-----------------|
| `read_file` | Read the contents of a file at a given path | `ALLOW_LOCAL_TOOLS=true` |
| `write_file` | Write or overwrite a file at a given path | `ALLOW_LOCAL_TOOLS=true` |
| `list_dir` | List files and directories at a given path | `ALLOW_LOCAL_TOOLS=true` |
| `apply_patch` | Apply a unified diff patch to a file | `ALLOW_LOCAL_TOOLS=true` |

See [File Tools](/tools/file-tools) for parameter schemas and examples.

## Shell Tool

## Security Notes

All tools that surface external data set `requires_sanitization = true`. This means their output passes through the Safety Layer (sanitizer + leak detector) before the LLM sees it. Tools that return only internal state (like `list_jobs` or `time`) do not require sanitization.

<Warning>
`shell` and file tools give the agent direct access to your filesystem and command environment. Only enable `ALLOW_LOCAL_TOOLS=true` on machines where you trust the agent to operate at that level.
</Warning>
